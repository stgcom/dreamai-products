## 3. Dashboard Structure

### 3.1 Daily Dashboard (Ops — viewed every morning)

**Owner:** Campaign operator / SOVEREIGN
**Format:** Slack/Discord summary card or Notion widget

| Section | Metrics | Anomaly Threshold |
|---|---|---|
| **Today's Pulse** | New leads, MQLs, calls booked, deals closed | ±30% vs 7-day rolling avg |
| **Ad Spend** | Yesterday's spend vs budget, CPA trending | Spend > 120% daily budget or CPA > 150% target |
| **Content Performance** | Yesterday's posts: impressions, engagement | Engagement rate < 50% of 30-day avg |
| **Pipeline Movement** | Deals advanced, proposals sent, won/lost | Any deal lost > $3K value |
| **Alerts** | Bounce rate spikes, unsub surges, ad disapprovals | Any single metric deviating > 2 std devs |

**Daily Checklist:**
- [ ] Check ad account spend pacing
- [ ] Review overnight lead flow

## 4. Experiment Framework

### 4.1 Experiment Categories

| Priority | Category | Impact Potential | Effort | Examples |
|---|---|---|---|---|
| **P0** | Offer & Pricing | Very High | Medium | Core offer structure, pricing tiers, guarantee |
| **P1** | Conversion (LP + CTA) | High | Low | Landing page copy, CTA placement, form length |
| **P2** | Channel & Creative | High | Medium | Ad creative, audience targeting, platform mix |
| **P3** | Nurture & Follow-up | Medium | Low | Email sequence order, cadence, subject lines |
| **P4** | Social & Content | Medium | Low | Post format, hook variations, posting time |

### 4.2 A/B Test Protocol

**Step 1: Hypothesis**
```
We believe [CHANGE] will result in [OUTCOME] because [REASON].
We'll know this is true when [METRIC] moves by [MINIMUM EFFECT SIZE]
with [CONFIDENCE LEVEL]% confidence.
```

**Step 2: Test Design**

| Parameter | Standard | Notes |
|---|---|---|
| **Minimum sample size** | 100 conversions per variant (or 2 weeks, whichever is longer) | For binary outcomes (convert/didn't) |
| **Statistical significance** | 95% confidence (p ≤ 0.05) | Use two-tailed test |
| **Practical significance** | Minimum 10% relative improvement | Don't ship winners that move < 10% |
| **Test duration** | Minimum 7 days, full business cycle | Covers weekday/weekend variance |
| **Secondary metrics** | Track but don't gate on | Engagement, bounce rate, time on page |
| **Guardrail metrics** | Must not degrade | Unsubscribe rate, brand sentiment |

**Step 3: Calculation Helper**

For binary metrics (conversion rate), minimum sample per variant:

| Baseline Conv Rate | Min Detectable Effect (20% relative) | Samples per Variant |
|---|---|---|
| 2% | 2.4% | ~7,600 |
| 5% | 6.0% | ~2,900 |
| 10% | 12.0% | ~1,300 |
| 20% | 24.0% | ~600 |
| 50% | 60.0% | ~165 |

### 4.3 Experiment Sequencing (Priority Roadmap)

**Month 1 — Foundation Tests**
| # | Test | Hypothesis | Priority | Duration |
|---|---|---|---|---|
| 1 | Lead magnet format: PDF vs Interactive tool | Interactive tools convert 25% better than PDFs | P1 | 2 weeks |
| 2 | Landing page: Long-form vs Short-form | Long-form (+ case study) increases conv rate by 15% for $3K+ offers | P1 | 2 weeks |
| 3 | Email subject line: Personalized vs Curiosity | Personalized subject lines lift open rate by 12% | P3 | 1 week |
| 4 | CTA: "Book a call" vs "Get a free assessment" | Value-framed CTA increases booking rate by 10% | P1 | 2 weeks |

**Month 2 — Channel Optimization**
| # | Test | Hypothesis | Priority | Duration |
|---|---|---|---|---|
| 5 | Ad creative: Video testimonial vs Problem-agitate | Video testimonials reduce CPA by 20% | P2 | 2 weeks |
| 6 | Audience: Job title targeting vs Interest-based | Job title targeting produces 30% higher MQL rate | P2 | 2 weeks |
| 7 | Email cadence: 3-day vs 5-day interval | 5-day intervals reduce unsubscribes by 40% without impacting conv | P3 | 2 weeks |
| 8 | Referral offer: Discount vs Exclusive access | Exclusive access drives 2x more referrals than discounts | P4 | 3 weeks |

**Month 3 — Scale & Pricing**
| # | Test | Hypothesis | Priority | Duration |
|---|---|---|---|---|
| 9 | Pricing page: Tiered vs Custom quote | Tiered pricing increases proposal acceptance by 15% | P0 | 3 weeks |

### 4.4 Experiment Log Template

| Field | Description |
|---|---|
| **Experiment ID** | EXP-001 |
| **Name** | Short descriptive name |
| **Category** | Offer / LP / Channel / Nurture / Content |
| **Priority** | P0–P4 |
| **Hypothesis** | Structured hypothesis statement |
| **Metric(s)** | Primary + secondary + guardrail |
| **Baseline** | Current performance |
| **MDE** | Minimum detectable effect |
| **Variant A** | Control |
| **Variant B** | Treatment |
| **Start Date** | — |
| **End Date** | — |
| **Sample Size** | Total + per variant |
| **Results** | Outcome for each variant |
| **P-value** | Statistical significance |
| **Winner** | A / B / Inconclusive |
| **Decision** | Ship / Iterate / Kill |
| **Learning** | Key insight for future tests |
| **Next Test** | Follow-up experiment inspired by this result |
